# Contributing — Ledger (expense-tracker-app)

Thanks for considering contributing! This project is small and friendly. Below are simple guidelines to make your contributions easy to review.

How to contribute
1. Fork the repository and create a feature branch: `git checkout -b feat/my-feature`
2. Keep changes focused and small. One feature or fix per pull request.
3. Update or add documentation under `docs/` for behaviour changes.
4. Open a pull request describing the change and the motivation.

Code style
- Keep the code readable and well-commented — this project purposefully keeps logic in a single file.
- Prefer clarity over cleverness. Use descriptive variable and function names.

Testing
- There are no automated tests in this starter. When adding features, include manual test steps in the PR description.

Issues
- If you find a bug or have a feature request, open an issue with steps to reproduce, expected behaviour, and screenshots if helpful.

Security
- If you discover a security vulnerability, please disclose it privately by opening an issue marked as `security` or contact the repository owner.
